사용자가 지정한 파일 또는 최근 변경된 코드를 리뷰합니다.

## 리뷰 기준

### 1. 동작 여부 (최우선)
- 코드가 의도대로 동작하는지
- 에러가 발생할 수 있는 부분이 있는지

### 2. 프로젝트 정합성
- CLAUDE.md의 Code Rules를 따르고 있는지
- ADR에서 결정한 기술 스택과 일치하는지
- 파이프라인(F1~F6)에서 이 코드의 위치와 역할이 명확한지

### 3. 코드 품질
- 변수/함수 이름이 의미를 잘 전달하는지
- 불필요하게 복잡한 부분이 없는지
- 보안 취약점(SQL 인젝션, 환경변수 노출 등)이 없는지

### 4. 학습 포인트 (입문자 지원)
- 사용된 패턴이나 라이브러리에 대해 간단히 설명
- "왜 이렇게 하는지" 배경 지식 제공
- 더 나은 방법이 있다면 비교하여 설명

## 출력 형식
- 문제점은 심각도별로 분류 (치명적 / 개선 권장 / 참고)
- 각 항목에 구체적인 수정 제안 포함
- 마지막에 전체 요약 한 줄
