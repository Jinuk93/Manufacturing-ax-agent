# 데이터셋 전략 (Dataset Strategy)

**상태:** Draft (검토 중)
**최종 수정일:** 2026-03-08

이 문서는 프로젝트에 필요한 모든 데이터셋을 기능(F1~F6)별로 정리하고,
각 데이터의 확보 방법(공공 데이터 / 합성 생성 / 미정)을 명시합니다.

---

## 1. 데이터셋 전체 맵

| 기능 | 필요 데이터 | 도메인 | 확보 방법 | 상태 |
|------|-------------|--------|-----------|------|
| F1, F2 | CNC 센서 데이터 (SCADA) | CNC 가공 | KAMP 공공 데이터 | 확보 방안 확정 |
| F3 | 작업 지시 내역 (MES) | CNC 가공 | 합성 생성 (ADR-002) | 설계 필요 |
| F3 | 부품 재고 현황 (ERP) | CNC 가공 | 합성 생성 (ADR-002) | 설계 필요 |
| F4 | 설비-고장-부품-매뉴얼 관계 (Ontology) | CNC 가공 | 미정 | ADR 필요 |
| F4 | 정비 매뉴얼 텍스트 | CNC 가공 | 미정 | ADR 필요 |

---

## 2. 공공 데이터: KAMP CNC 센서 데이터

### 2.1. 데이터 개요

- **출처:** KAMP (한국 AI 제조 플랫폼) 공공 데이터셋
- **대상 설비:** CNC 기계 가공 설비
- **용도:** F1(전처리), F2(이상 탐지 + Forecasting)

### 2.2. 필요한 데이터 항목 (EDA에서 검증 필요)

| 항목 | 용도 | 비고 |
|------|------|------|
| 진동(Vibration) | 이상 탐지, 예측 | 핵심 센서 |
| 전류(Current) | 부하 상태 판별 | 핵심 센서 |
| 온도(Temperature) | 과열 감지 | 보조 센서 |
| 주축 속도(Spindle Speed) | 가공 조건 맥락 | 보조 센서 |
| Timestamp | 시계열 정렬 기준 | 필수 |

> **주의:** 위 항목은 가설입니다. Phase 1 (Data Exploration)에서 실제 KAMP 데이터를
> 다운받아 확인한 후 수정합니다.

### 2.3. 확보 방법

- KAMP 홈페이지에서 무료 회원가입 후 다운로드
- CSV 형태로 제공될 것으로 예상 (EDA에서 확인)

---

## 3. 합성 데이터: MES (작업 지시)

### 3.1. 합성이 필요한 이유

실제 MES 데이터는 기업 내부 1급 비밀로, 외부 공개 데이터셋이 전무함. (ADR-002)

### 3.2. 필요한 데이터 항목

| 항목 | 설명 | 예시 값 |
|------|------|---------|
| work_order_id | 작업 지시 번호 | WO-20260301-001 |
| equipment_id | 설비 ID (KAMP와 연결 키) | CNC-001 |
| product_name | 생산 중인 제품명 | 알루미늄 플랜지 A-300 |
| order_quantity | 주문 수량 | 500 |
| completed_quantity | 현재까지 완료 수량 | 320 |
| due_date | 납기일 | 2026-03-15 |
| priority | 납기 중요도 | 긴급 / 일반 / 여유 |
| status | 작업 상태 | 진행중 / 대기 / 완료 |
| start_time | 작업 시작 시간 | KAMP 타임라인에 맞춤 |

### 3.3. 합성 시 핵심 원칙

- KAMP 센서 데이터의 타임라인(Timestamp)에 정확히 맞물려야 함
- 비즈니스 시나리오가 의미 있어야 함 (아래 §6 참조)

---

## 4. 합성 데이터: ERP (부품 재고)

### 4.1. 필요한 데이터 항목

| 항목 | 설명 | 예시 값 |
|------|------|---------|
| part_id | 부품 번호 | PART-BRG-001 |
| part_name | 부품명 | 주축 베어링 7210C |
| equipment_id | 대상 설비 ID | CNC-001 |
| stock_quantity | 현재 재고 수량 | 3 |
| min_stock | 안전 재고 기준 | 2 |
| lead_time_days | 발주 후 입고 소요일 | 7 |
| unit_price | 단가 (원) | 150000 |
| last_replaced | 최근 교체일 | 2026-02-10 |

### 4.2. 합성 시 핵심 원칙

- 설비별 필수 교체 부품 목록이 온톨로지(F4)와 일관되어야 함
- 재고가 부족한 시나리오와 충분한 시나리오를 모두 포함해야 함

---

## 5. 미정 데이터: 온톨로지 & 정비 매뉴얼

### 5.1. 온톨로지 관계 데이터

Neo4j에 적재할 '설비 -> 고장 코드 -> 필요 부품 -> 조치 매뉴얼' 관계망.

**확보 방안 후보:**
- (A) CNC 장비 제조사 공개 매뉴얼에서 관계 추출
- (B) CNC 도메인 지식 기반으로 직접 설계 후 합성
- (C) 기존 산업 표준 고장 코드 체계(ISO 등) 참고하여 구성

> **결정 시점:** Phase 1 (Data Exploration) 이후, ADR로 기록

### 5.2. 정비 매뉴얼 텍스트

RAG 검색 대상이 될 매뉴얼 문서.

**확보 방안 후보:**
- (A) CNC 장비 제조사의 공개 기술 문서 수집
- (B) CNC 정비 관련 공개 자료(논문, 기술 블로그 등) 활용
- (C) 도메인 지식 기반으로 합성 매뉴얼 직접 작성

> **결정 시점:** Phase 1 (Data Exploration) 이후, ADR로 기록

---

## 6. 합성 데이터 비즈니스 시나리오 (설계 기준)

합성 데이터가 의미 있으려면, LLM이 **서로 다른 판단**을 내릴 수 있는
다양한 비즈니스 상황을 커버해야 합니다.

### 시나리오 후보

| # | 시나리오 | MES 조건 | ERP 조건 | 기대 LLM 판단 |
|---|---------|----------|----------|---------------|
| S1 | 긴급 납기 + 부품 재고 충분 | 납기 임박, 잔량 많음 | 재고 있음 | 즉시 정지 후 부품 교체, 빠르게 재가동 |
| S2 | 긴급 납기 + 부품 재고 부족 | 납기 임박, 잔량 많음 | 재고 없음 | 속도 감속 운전, 긴급 발주 |
| S3 | 여유 납기 + 부품 재고 충분 | 납기 여유, 잔량 적음 | 재고 있음 | 계획 정비 스케줄링 |
| S4 | 여유 납기 + 부품 재고 부족 | 납기 여유, 잔량 적음 | 재고 없음 | 발주 후 입고 대기, 계획 정비 예약 |

> **주의:** 위 시나리오는 초안입니다. 합성 데이터 생성 전에 확정하며, ADR로 기록합니다.

---

## 7. 데이터 간 연결 관계

```
KAMP 센서 (실데이터)
    |
    |-- equipment_id + timestamp 기준 JOIN
    |
    +-- MES 작업 지시 (합성) ---- due_date, priority
    |
    +-- ERP 부품 재고 (합성) ---- stock_quantity, lead_time
    |
    +-- Neo4j 온톨로지 (미정) --- 고장코드 → 부품 → 매뉴얼
```

모든 데이터는 `equipment_id`와 `timestamp`를 기준으로 연결되어야 하며,
이것이 F3(Data Alignment)의 핵심 로직입니다.
