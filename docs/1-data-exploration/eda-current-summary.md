# Phase 1 EDA 현재 요약

최종 갱신: 2026-03-10

## 지금까지 무엇을 했나

- Kaggle CNC Mill EDA 노트북의 첫 실행 셀을 구성했습니다.
- 데이터 경로와 파일 존재 여부를 확인했습니다.
- `train.csv`를 읽어 실험 메타데이터와 라벨 분포를 확인했습니다.
- `experiment_01.csv` ~ `experiment_18.csv`의 구조가 서로 같은지 확인했습니다.
- 전체 실험을 합쳐 상수 컬럼과 0값 고정 컬럼 후보를 찾았습니다.
- `experiment -> equipment_id`, `sequence -> timestamp` 합성 초안을 노트북 안에서 바로 확인할 수 있게 만들었습니다.

## 쉽게 말하면 지금 확인한 것

### 1. 데이터는 정상적으로 읽힌다

- Kaggle CNC Mill 실험 파일 18개를 모두 찾았습니다.
- `train.csv`도 정상적으로 읽혔습니다.

### 2. 라벨 구조는 예상과 같다

- `train.csv` 크기: 18행, 9컬럼
- 공구 상태 분포:
  - `worn`: 10개 실험
  - `unworn`: 8개 실험

### 3. 실험 파일 구조는 일관적이다

- 18개 실험 CSV의 컬럼 구조가 모두 동일했습니다.
- 즉, 한 번 전처리 규칙을 정하면 전체 실험에 같은 방식으로 적용할 수 있습니다.

### 4. 바로 제거 후보가 보인다

- 통합 데이터 크기: 25,286행, 52컬럼
- 상수 컬럼 5개 확인
- 0값 고정 numeric 컬럼 4개 확인

현재 바로 눈에 보이는 주요 후보:

- `Z1_CurrentFeedback`
- `Z1_DCBusVoltage`
- `Z1_OutputCurrent`
- `Z1_OutputVoltage`
- `S1_SystemInertia`

의미:

- Z축 전기 피드백 4개는 전체 데이터에서 계속 0이라서 분석 가치가 낮을 가능성이 큽니다.
- `S1_SystemInertia`는 상수라서 모델 입력으로는 거의 의미가 없습니다.

### 5. 내부 표준 형식 초안도 시작했다

노트북에서 임시로 아래 규칙을 적용해 preview 했습니다.

- `experiment_01` -> `CNC-001`
- `experiment_02` -> `CNC-002`
- `sequence * 100ms` -> `timestamp`

이건 아직 확정 규칙은 아니고, Phase 1 EDA에서 검증해 가면서 다듬을 초안입니다.

## 왜 이 작업이 중요한가

이 단계는 아직 "모델을 만든다" 단계가 아닙니다.

먼저 확인해야 하는 것은 아래입니다.

- 실제로 어떤 컬럼이 쓸 만한가?
- 버려야 할 컬럼은 무엇인가?
- 공개 데이터에 없는 `timestamp`, `equipment_id`를 어떤 규칙으로 만들 것인가?

이걸 먼저 정리해야 다음 단계에서:

- 전처리 기준
- 이상탐지 입력 컬럼
- forecasting 입력 컬럼
- Canonical Model 초안

을 흔들리지 않고 정할 수 있습니다.

- 전력

이 중에서 worn / unworn 차이가 보이는지 먼저 확인합니다.

- 실제로 완전히 버릴 컬럼 목록을 정리합니다.
