현재 프로젝트 상태를 점검합니다.

## 점검 절차

1. **현재 Phase 확인**
   - README.md의 Phases 표에서 "진행중" 상태인 Phase 확인

2. **해당 Phase 폴더 점검**
   - docs/[현재 Phase 폴더]/ 안의 문서 목록 확인
   - 각 문서의 완성도 평가 (빈 파일, TODO 남은 부분 등)

3. **관련 미결사항 점검**
   - open-items.md에서 현재 Phase에서 결정해야 할 항목 확인
   - Open 상태인 항목이 있으면 목록으로 제시

4. **Phase 완료 기준 체크**
   Phase 0: PRD 확정, 데이터셋 전략 확정, 미결사항 정리 완료
   Phase 1: KAMP EDA 완료, 센서 컬럼 확정, 합성 데이터 설계 기준 확정
   Phase 2: 세부 파이프라인 확정, DB 스키마 확정, 기술 검증 완료

5. **결과 보고**
   - 완료된 항목 / 미완료 항목 표로 정리
   - 다음 Phase로 넘어갈 수 있는지 판단
   - 넘어가려면 남은 작업 목록 제시
