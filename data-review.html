우리가 해결하려는 문제

CNC 공장에서 실제로 겪는 3가지 문제. 이걸 해결하려면 데이터가 필요해요.

1. 이상탐지

장비 센서가 평소와 달라지고 있는데, 사람이 눈치채기 전에 시스템이 먼저 감지해야 해요.

예를 들어 X축 전류가 평소 0.08A인데 갑자기 0.3A로 튀면 — "뭔가 이상하다"고 자동으로 알려주는 거예요.

2. 예지보전

이상이 감지되면, "앞으로 어떻게 될지" 예측해서 지금 멈춰야 하는지, 감속하면 되는지, 다음 정비 때 하면 되는지 판단해야 해요.

단순히 "이상하다"를 넘어서, "30분 뒤에 위험해진다" 같은 미래 예측이 핵심이에요.

3. 지식 구조화 (온톨로지)

"이 고장코드가 뜨면 → 이 부품을 교체하고 → 매뉴얼 3장을 참고해"

이런 정비 지식이 지금은 숙련공 머릿속에만 있어요. 이걸 그래프(온톨로지)로 구조화해서 AI가 검색하고 활용할 수 있게 해야 해요.

이 3가지를 합치면 최종 모습은 이거예요:

센서 이상 감지 → 원인 추정 → 매뉴얼 검색 → 조치 보고서 자동 생성

사람이 "뭐 해야 하지?" 가 아니라, 시스템이 먼저 알려주는 거예요.

"유효 42컬럼"은 뭔 뜻이야?

원래 48개 컬럼인데, 분석해보니 6개가 쓸모없어요.

상수 컬럼 2개 — 값이 항상 같아서(12.0, 1.0) 분석 의미 없음.

Z축 전기 피드백 4개 — 18개 실험 전부에서 값이 0. 센서가 연결 안 됐거나 수집되지 않은 것.
(단, Z축의 위치/속도/가속도는 정상! 전기 데이터만 0이에요.)

48 - 2(상수) - 4(Z축 전기) = 42개가 실제 분석에 쓸 수 있는 컬럼.
